# Mobile Testing

## Coverage: 80% minimum

## TDD Workflow (MANDATORY)
1. Write test first (RED)
2. Run test - should FAIL
3. Implement minimal code (GREEN)
4. Refactor (IMPROVE)
5. Verify 80%+ coverage

## Test Types
- Unit: ViewModels, UseCases
- Integration: Repositories, Flows
- UI: Compose with Espresso

## Agents
- `mobile-tdd-guide` - TDD enforcement
- `mobile-e2e-runner` - E2E tests
